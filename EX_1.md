Bài tập: SS02_HW01
1. Đáp án lựa chọn
Khẳng định rõ ràng: Chọn "Phương án C".

2. Phân tích phương án C
Phương án C là sự kết hợp hoàn hảo để tối ưu năng suất làm việc của lập trình viên trong khoảng thời gian rất ngắn. Cụ thể:

Web Chat phù hợp nhất để soạn tài liệu dài như tài liệu hướng dẫn sử dụng API (API Guide) nhờ vào khả năng lập luận ngôn ngữ và trình bày xuất sắc.
Agentic Assistant có phạm vi hiểu ngữ cảnh toàn dự án (Workspace Context), nên là lựa chọn duy nhất và tốt nhất để tự động cập nhật đồng bộ cấu trúc lớp Book trên 4 tệp tin (Book.java, BookService.java, BookController.java, và BookRepository.java) cùng lúc mà không cần thao tác thủ công.
Inline Code Assistant giải quyết lỗi cú pháp (Quick Fix) ngay tại dòng code trong IDE, giúp lập trình viên sửa lỗi nhanh chóng mà không bị gián đoạn luồng suy nghĩ.
3. Nhược điểm của các phương án khác
Phương án A (Chỉ dùng Web Chat): Việc sao chép liên tục các đoạn code và tệp tin từ IDE sang trình duyệt web để sửa đổi sẽ gây ra hiện tượng Chuyển đổi ngữ cảnh (Context Switching) nghiêm trọng. Điều này không chỉ gây tốn thời gian mà còn rất dễ dẫn đến nhầm lẫn khi cập nhật lại vào dự án.
Phương án B (Chỉ dùng Inline Code Assistant): Công cụ này giới hạn phạm vi ngữ cảnh ở tệp đang mở và một số tệp liên quan gần nhất. Do đó, nó không thể tự động mở và sửa đa tệp tin (4 file cùng lúc) hoặc viết một tài liệu có độ dài quá dài.
